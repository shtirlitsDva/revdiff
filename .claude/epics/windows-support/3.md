# Task: PowerShell build and test scripts

## Description

The current `Makefile` calls `cp`, `grep -v`, and writes to `.bin/revdiff` with no `.exe` suffix ‚Äî none of this works in PowerShell or `cmd.exe`. Add `build.ps1` and `test.ps1` at the repo root that mirror `make build` and `make test`. The Makefile is **not** modified; Unix contributors continue using `make`.

## Acceptance Criteria

- [ ] `build.ps1` at repo root, runnable from a PowerShell prompt at the repo root via `.\build.ps1`.
- [ ] `build.ps1` produces `.bin\revdiff.exe` from `cmd\revdiff` using the same `go build` flags the Makefile uses (currently: trimpath + ldflags injecting branch/version).
- [ ] `build.ps1` honors a `$env:VERSION` override the same way the Makefile honors `$(BRANCH)` ‚Äî defaults to the current git branch when unset.
- [ ] `build.ps1` exits non-zero on failure and prints the failing command.
- [ ] `test.ps1` at repo root, runnable via `.\test.ps1`.
- [ ] `test.ps1` runs `go test` over all packages with the same race detector + coverage flags as `make test`.
- [ ] `test.ps1` excludes mock files (`*_mock.go`) from coverage just like the Makefile does ‚Äî use `go list ./...` filtered in PowerShell, **not** `grep -v`.
- [ ] `test.ps1` exits non-zero on test failure.
- [ ] Both scripts use `Set-StrictMode -Version Latest` and `$ErrorActionPreference = 'Stop'`.
- [ ] Both scripts use `Join-Path` (or `[IO.Path]::Combine`) for path construction ‚Äî no hardcoded `\` separators.
- [ ] Both scripts include a header comment block describing what they do and pointing to their Makefile counterpart.
- [ ] `Makefile` is byte-identical to master ‚Äî no modifications.
- [ ] `make build && make test` on macOS/Linux still works unchanged.

## Technical Details

### Files to create
- `build.ps1` ‚Äî ~30 lines.
- `test.ps1` ‚Äî ~30 lines.

### Files to read (do not modify)
- `Makefile` ‚Äî read every target to confirm flag parity. Pay attention to how `BRANCH`, `VERSION`, and the ldflags string are constructed. The `.PHONY` declarations and target names don't matter to PowerShell, but the `go build` arguments do.

### Sketch

`build.ps1`:
```powershell
# build.ps1 ‚Äî Windows mirror of `make build` from the project Makefile.
# Outputs: .bin\revdiff.exe
# Usage: .\build.ps1   (optionally set $env:VERSION first)

Set-StrictMode -Version Latest
$ErrorActionPreference = 'Stop'

$repoRoot = $PSScriptRoot
$binDir   = Join-Path $repoRoot '.bin'
$out      = Join-Path $binDir 'revdiff.exe'

if (-not (Test-Path $binDir)) { New-Item -ItemType Directory -Path $binDir | Out-Null }

$version = if ($env:VERSION) { $env:VERSION } else { (& git rev-parse --abbrev-ref HEAD).Trim() }

Push-Location (Join-Path $repoRoot 'cmd\revdiff')
try {
    & go build -trimpath -ldflags "-X main.version=$version" -o $out
    if ($LASTEXITCODE -ne 0) { throw "go build failed with exit $LASTEXITCODE" }
} finally {
    Pop-Location
}

Write-Host "Built $out (version: $version)"
```

`test.ps1`:
```powershell
# test.ps1 ‚Äî Windows mirror of `make test` from the project Makefile.
# Runs go test with race detector + coverage, excludes *_mock.go from coverage.

Set-StrictMode -Version Latest
$ErrorActionPreference = 'Stop'

$packages = & go list ./... | Where-Object { $_ -notmatch '_mock(\.go)?$' }
& go test -race -coverprofile=coverage.out $packages
if ($LASTEXITCODE -ne 0) { throw "tests failed with exit $LASTEXITCODE" }
```
