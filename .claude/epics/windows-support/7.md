## Technical Details

### Validation environment
- Hardware/OS: Windows 11 (the author's machine).
- Terminal: WezTerm (latest stable, with `wezterm cli` available).
- Shell: PowerShell (5.1 or 7+, both should work for the launchers; 5.1 is the practical floor).
- Git for Windows installed and on `PATH`.
- Go matching `go.mod` installed and on `PATH`.
- Claude Code CLI installed for plugin testing.

### Validation procedure

1. Fresh clone the repo on the Windows machine.
2. Run `.\build.ps1` and confirm `.bin\revdiff.exe` exists.
3. Run `.\test.ps1` and confirm green.
4. Run `.\bin\revdiff.exe --dump-config` and copy the output into the validation log.
5. Walk every PRD success criterion in order. Tick boxes as you go.
6. For plugin testing, start a Claude Code session inside a WezTerm tab and invoke `/revdiff` and `/plan-review` (or whatever the skill triggers are).
7. Switch to macOS, fresh clone, run `make build && make test`, invoke the skills, confirm no regressions.
8. Diff `vendor/`, `go.mod`, `go.sum`, `Makefile`, and the four bash scripts against `master` to confirm no incidental changes.
9. Bump plugin version (after asking the user).
10. Open a final summary PR or commit that ties off the epic.

### Files to modify
- `.claude-plugin/plugin.json` — version field only.
- `.claude-plugin/marketplace.json` — version field only.
- (Possibly) `plugins/revdiff-planning/plugin.json` — if it exists and has its own version.

### Files to read (do not modify)
- All task files in this epic — confirm each is closed before bumping.
- The PRD — use it as the canonical success criteria checklist.

### Risks at validation time
- **bubbletea ConPTY rendering**: if backgrounds look wrong or box-drawing is garbled, check for an open issue at `charmbracelet/bubbletea` before patching locally.
- **`wezterm cli spawn` exit semantics**: if the spawned tab doesn't close cleanly, the launcher's wait/cleanup pattern needs another iteration — file as a sub-task of this validation, not a fresh epic.
- **Git for Windows line endings**: if `core.autocrlf=true` perturbs diff parsing, document the workaround (`git config --global core.autocrlf input` for revdiff repos) rather than coding around it.
